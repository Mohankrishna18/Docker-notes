# Docker-notes

-------------

1) What is Docker ?

An open-source platform/tool that lets you build, run, and manage containers.
- Provides the engine, CLI, and ecosystem (like Docker Hub) to handle container lifecycle.

2. Docker Image ?
- A blueprint that defines what goes inside a container (application code, libraries, dependencies, environment variables).
- Built from a Dockerfile and used to create containers.

3. What is container ?
- Docker is a package itself which includes each and everything with all the libraries with configuration with the application, everything combined together from a docker container and this docker container will be running in our docker environment.

4. How docker container will be created ?
- Docker container created on base of a docker image. Docker image is something that will consists of each and every configuration in each and everything.
- This docker image will live in the docker registry and from the docker registry what we can do is which ever environment we want this application to be running in that environment we can take this docker image from the docker registry and we can have this docker image here, this docker image we can run whenver we run this docker image that is called container.
- that will be the container which will have each and everything which is a running entity in the docker environment.

- Docker image consists of different layers and those layers combined together will make a single docker image and that docker image we can run and that is going to called my docker container.
